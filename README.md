## マルコフ連鎖とは
今の状態から次の状態に遷移するときに，それまでの状況がどう変化してきたかに関わらず，「現在の状況のみから」遷移確率が決定されるようなモデルです．

式で定義すると以下のようになります．

```math
P
\left\{
X_{t+1} = x | X_1, X_2,..., X_t 
\right\}
=P(X_{t+1}=x| X_t)
```
簡単に言うと，式中の$X_{t+1}$の値は$X_t$によってのみ決まり，$X_1...X_{t-1}$がいかなる値であろうと関係ないと言うことです．

これは1階のマルコフ連鎖(単純マルコフ連鎖)の式ですが，2手前までの状態を考慮する2階マルコフ連鎖など，高階のものも存在します．

## じゃんけんへ応用する際の定式化
大前提として，じゃんけんの手は{グー，チョキ，パー}
の3状態として定義できます．

プログラムで扱う都合上
```math
S=
\left\{
{0（グー），1（チョキ），2（パー）}
\right\}
```
と表現します．

次に，このプログラムの最も重要なポイントが遷移確率表現です．2次元の配列で表現しています．
```math
遷移確率 =
\begin{bmatrix}
P(グー|グー) & P(チョキ|グー) & P(パー|グー)\\
P(グー|チョキ) & P(チョキ|チョキ) & P(パー|チョキ)\\
P(グー|パー) & P(チョキ|パー) & P(パー|パー)
\end{bmatrix}
```

遷移確率の導出方法ですが，それまでのプレイヤーが出した手の累積から求められます．
したがって確率分布への変換式は，
```math
P_{ij} = P(X_{t+1}=j|X_t) = \frac{C_{ij}}{\Sigma_{k=0}^2C_{ik}}
```
ただし，
```math
i, j \in 
\left\{グー,チョキ,パー\right\}
\\
C_{i,j}=手iの後にjが出た回数
```
です．
確率分布が定まると，そこからプレイヤーの出す手が予測されます．
方針は2種類あり，
* $P_{x_t, j}(k=0,1,2)$のうち条件付き確率が最も高い手を決定的に予測手とする方法
```math
\hat{x_{t+1}} = \arg\max_{j}P_{x_t, j}
```
* 分布を元に確率的に選ぶ方法
```math
X_{t+1} 〜 P_{x_t,j}
```
今回は後者を採用してみました。

## 実装の話
### `marcov_model.py`
上記の定式化をそのままPythonで実装したものです．

注意点は，二次元リスト`distribution`は単なる遷移して出た手の累計で，厳密には確率分布ではありません．

上の説明では，累計から確率分布への変換を行なっていますが，実装の上ではわざわざ $\Sigma_{k=0}^2C_{ik}$ で割る必要はないのです．
代わりに`random.randint`の範囲を$(0, \Sigma_{k=0}^2C_{ik})$にしています．

その他は定式化した通りに計算するだけです．
このコードでは，予測と分布の更新の間にプレイヤーの実際に入力を挟んでいます．
* プレイヤーの手の予測 (17-31行目) \
直前の状態 `prev` に対して`distribution[prev]`をもとに次の手を
`r = random.uniform(0, s)` は$(0〜prevの合計)$の範囲で等しい確率の浮動小数を返します．その後の条件分岐では`distribution[prev][k]`の値が大きいほど，その条件の範囲が広くなるため成立しやすくなります．
したがって結果的に$P_{i,j}$の確率分布に従います．

* 分布の更新\
実際のプレイヤーの入力が行われると，`prev` → `player`の遷移を記録します．ここはシンプルに`x[pev][player]`に+1するだけです．

#### (余談1)
じゃんけんの勝敗判定ロジックで，二人の差を3で割った余りが使われるのは有名な話です．

(グー，チョキ，パー) = (0, 1, 2)とします．
```
(A - B) % 3 == 0 :あいこ
(A - B) % 3 == 1 :Bの勝ち
(A - B) % 3 == 2 :Aの勝ち
```
コンパクトに書けるので実装でも便利ですね．
作ってて思い出したので書いておきました．

#### (余談2)
最初のマルコフ連鎖の説明で触れましたが，実はこの場合においても2階以上のマルコフ連鎖は存在し，実装もそこまで難しくありません．

イメージ的には，2階マルコフ連鎖なら「直前と2個前の手から次の手を予測する」というように，数個前の状態も考慮するということです．

ただしやってみれば分かりますが（`distribution`のサイズを大きくするだけ），階数が増えるほど強いモデルになるというわけでもないのです．リストのサイズが大きくなるということは，十分なデータ数で満たされるのが遅くなり，適応が遅くなることを意味します．

## パーセプトロンとは
パーセプトロンは，ニューラルネットワークの古典的手法と言われ，入力に対する重みづけの計算を行うことによって，分類結果を返します．

基本の式は次のとおりです．
```math
v = \begin{bmatrix}w_1, w_2, ..., w_n\end{bmatrix}

\begin{bmatrix}
x_1 \\
x_2 \\
... \\
x_n
\end{bmatrix}
+b
```
ただし
* $x$：入力ベクトル
* $w$：重みベクトル
* $b$：バイアス（線形式の切片に当たる）

です．
行列で書くとハードル高くなりますがやってることは一次関数です．
```math
v = w_1x_1+w_2x_2+...+w_3x_3+b
```
における$x_{1,2,...,n}$の係数$w_{1,2,...,n}$を学習で決めていきます．

## じゃんけんへ応用する際の定式化
マルコフ連鎖の時同様，じゃんけんの手は
```math
S=
\left\{
{0（グー），1（チョキ），2（パー）}
\right\}
```
と表現します．
ただしパーセプトロンの場合は，さらに扱いやすくするため 0/1ベクトルに変換します．
```math
グー = (1,-1,-1), \
チョキ = (-1,1,-1), \
パー = (-1,-1,1), \
```

通常の単純パーセプトロンよりも複雑である点が，ここでは過去にプレーヤーが出した手を時系列とみなしていて，直近$N$回の履歴を扱います．
```math
x = (x_t, x_{t-1}, ... x_{t-N+1})
```
したがって$x$は$N\times3$のバイナリ行列と見なすことができます．

そして，出力である$v$は3つの手それぞれのスコアを計算します．2クラス分類の場合は$v$は1つの変数ですが，じゃんけんの予測は他クラス分類に該当するため，$v$は3次元ベクトルとなります．

重み$w$は，入力$x$と出力$w$を結びつけているため，そのサイズは$x$と$v$によって定まります．
$x$が$3 \times N$, $v$が$3$であるため$w$は$3\times N \times3$の行列で表すことができます．
